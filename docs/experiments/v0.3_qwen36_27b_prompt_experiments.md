# Baseline v0.3 qwen36-27b Prompt Experiments

Baseline v0.2 is frozen before this experiment series begins. Its registered run selection, sanitized per-task artifacts, consistency audit, failure matrix, and dashboard remain reference data and must not be overwritten by v0.3 results.

## Research Question

Baseline v0.3 compares prompt strategies on qwen36-27b only. It is a prompt experiment, not a model comparison. qwen36-27b remains the fixed quality reference so changes can be attributed to prompt behavior under otherwise matched settings.

The available profiles in `configs/prompt_profiles.yaml` are:

- `neutral_baseline`: preserves the historical system prompt.
- `strict_rtl_only`: requests synthesizable RTL with no prose, Markdown, or unsolicited testbench.
- `low_area_low_power`: prioritizes area and power while treating timing as a constraint, without making unsupported PPA claims.
- `no_explanation_code_only`: applies the strictest output formatting for code-extraction reliability.

When `--prompt-profile` and `prompt.profile` are both absent, RTLBench uses the unchanged historical system prompt. Selecting `neutral_baseline` makes that same prompt explicit in the run metadata.

## Smoke First

The smoke helper prints all 12 benchmark/profile commands by default and does not execute them:

```bash
python scripts/run_v0.3_prompt_smoke.py --base-url "$OPENAI_BASE_URL"
```

Review the printed commands, then opt in to the same small runs:

```bash
python scripts/run_v0.3_prompt_smoke.py --base-url "$OPENAI_BASE_URL" --execute
```

Use `--benchmarks` and `--profiles` with comma-separated names to run a smaller matrix. `--workers`, `--limit`, and `--notes-prefix` override the smoke execution settings without changing the configs.

After runs finish, generate the smoke-only report:

```bash
python scripts/summarize_v0.3_prompt_smoke.py
```

The summarizer reads only `summary.json` and `run_metadata.json`. Missing combinations remain explicitly marked as missing. Review `reports/v0.3_qwen36_27b_prompt_smoke_summary.md` before scheduling any full benchmark run.

## Run Isolation

Use a distinct base output directory for every benchmark and prompt profile:

```text
outputs/experiments/v0.3_prompt_profiles/<benchmark>/<prompt_profile>/<benchmark>/<model>/<timestamp>/
```

The final benchmark/model/timestamp components are added by the existing runner. This keeps current output behavior backward compatible. Never write experiment results into baseline v0.1 or v0.2 artifact/report paths.

## Comparison Rules

Keep the benchmark version, task subset, qwen36-27b endpoint, temperature, top-p, max tokens, samples per task, timeout, and evaluator identical across prompt profiles. Record the selected profile in `config_snapshot.yaml`, `run_metadata.json`, and `report.md`.

Reuse the existing pass/fail metrics, sanitized per-task export, cross-model failure analysis machinery, and dashboard flow in experiment-specific paths. Treat `low_area_low_power` as a prompt hypothesis only: exact PPA claims require synthesis reports and functional equivalence evidence.

No full benchmark run should begin until the smoke summary has been reviewed for matched settings, successful evaluation, plausible failure categories, and complete metadata.

## Larger Validation Scaffold

The candidate decision report selects only these profiles for the next larger validation slice:

- `neutral_baseline`
- `strict_rtl_only`

The held-back profiles, `low_area_low_power` and `no_explanation_code_only`, are not included in the larger scaffold.

Preview the larger validation commands without model calls:

```bash
python scripts/run_v0.3_prompt_larger.py --base-url "$OPENAI_BASE_URL"
```

This prints six commands:

- VerilogEval, limit 20, for both selected profiles
- RTLLM, limit 10, for both selected profiles
- RTL-OPT equivalence, limit 10, for both selected profiles

The larger output root is separate from smoke:

```text
outputs/experiments/v0.3_prompt_profiles_larger/<benchmark>/<prompt_profile>/<benchmark>/<model>/<timestamp>/
```

Execute later only after reviewing the dry run:

```bash
python scripts/run_v0.3_prompt_larger.py --base-url "$OPENAI_BASE_URL" --execute
```

Summarize only after real larger-validation runs exist:

```bash
python scripts/summarize_v0.3_prompt_larger.py
```

If no finished larger runs are found, the summarizer prints a missing-run notice and does not create `reports/v0.3_qwen36_27b_prompt_larger_summary.md` or `reports/v0.3_qwen36_27b_prompt_larger_summary.csv`.

Do not mix larger-validation outputs with smoke outputs or Baseline v0.1/v0.2 artifacts. Larger results are still prompt-experiment data, not final proof of prompt superiority.
