# Feature Spec: v0.7 RFID-APBench Release Closeout

## Goal

Close out the v0.7 RFID-APBench public/synthetic benchmark cycle after the post-fix expanded baseline and failure analysis.

The v0.7 post-fix expanded baseline is complete and interpretable: 30/30 responses were non-empty, 30/30 extracted, 27/30 passed functional simulation, and 27/30 produced valid area/activity scores. The only remaining invalid rows were concentrated in `ap_006_wakeup_edge_filter` and have been classified as candidate implementation failures rather than endpoint, token-limit, extractor, tool, evaluator, scoring, or benchmark asset defects.

This closeout is documentation and release planning only. It is not another benchmark run, model comparison, prompt-profile experiment, benchmark modification, private evaluation, fine-tuning, or training-data creation.

## Required Scope

Create a sanitized release closeout report that:

1. Summarizes the v0.7 goals and fixes.
2. Summarizes the thinking-disabled request propagation fix.
3. Summarizes the `ap_001_idle_counter` prompt clarity update and validation result.
4. Summarizes the post-fix expanded 10-task baseline result.
5. Summarizes the `ap_006_wakeup_edge_filter` failure analysis and limitation.
6. States that response-boundary reliability is closed for this v0.7 evidence: no empty/null-content/token-limit rows in the 30-row baseline.
7. States that benchmark assets remain unchanged after the approved `ap_001` prompt clarity update.
8. States that `completion_reliability` remains rejected/unused.
9. States that global max-token increase is not justified.
10. States that fine-tuning remains out of scope and not recommended.
11. States whether v0.7 is ready to tag/release as a public benchmark milestone.
12. Recommends exactly one next bounded direction.

## Decision Rules

Recommend exactly one:

- `v0.8_benchmark_hardening_plan_spec` if v0.7 is ready and the next cycle should harden benchmark/reporting/automation.
- `ap006_targeted_validation_spec` if v0.7 should not close until more `ap_006` evidence is gathered.
- `release_blocker_bugfix_spec` only if a concrete blocker or repository defect is found during closeout.
- `no_immediate_followup_beyond_documentation` if v0.7 is complete and no next planning spec is needed.

Do not recommend fine-tuning, private evaluation, model comparison, max-token increase, prompt-profile experiments, benchmark modification, broad rerun, or training-data creation from this closeout alone.
