# Feature Spec: v0.5 RFID-APBench Tool-Enabled Reference Run

## Goal

Run the RFID-APBench reference validation in an environment where Icarus Verilog (`iverilog` + `vvp`) and Yosys are available, so the five public/synthetic reference tasks produce real sanitized compile, simulation, activity, synthesis, and area results.

The previous reference-validation implementation correctly handled missing tools and produced a sanitized blocker report, but it did not validate the references because all required tools were unavailable.

This stage is still reference-only. Do not run model jobs.

## Inputs

Read first:

- `docs/specs/v0.5-rfid-apbench-reference-validation.md`
- `scripts/validate_rfid_apbench_references.py`
- `reports/v0.5_rfid_apbench_reference_validation.md`
- `reports/v0.5_rfid_apbench_reference_validation.csv`

## Required Scope

1. Ensure the validation environment has `iverilog`, `vvp`, and `yosys` available on PATH.
2. Run the existing reference validation script against `benchmarks/rfid_apbench`.
3. Regenerate sanitized Markdown/CSV reports.
4. If all five references compile, simulate, emit VCD activity, and synthesize, optionally write sanitized per-task `expected/reference_metrics.yaml` files.
5. If any task fails, fix only public/synthetic task fixtures or script bugs needed for reference validation.
6. Keep raw VCDs, logs, compiled artifacts, and synthesis scratch files under `.tmp/` only.

## Command

Run:

```bash
python scripts/validate_rfid_apbench_references.py \
  --benchmark-root benchmarks/rfid_apbench \
  --work-dir .tmp/rfid_apbench_reference_validation \
  --output-md reports/v0.5_rfid_apbench_reference_validation.md \
  --output-csv reports/v0.5_rfid_apbench_reference_validation.csv \
  --write-reference-metrics
```

Then run:

```bash
python -m pytest
```

## Required Report Outcome

The Markdown and CSV reports must show per task:

- compile_status
- simulation_status
- activity_status
- synthesis_status
- area_status
- timing_status
- reference_area when available
- area_unit when available
- reference_activity when available
- activity_metric when available
- failure_category
- notes/blocker

Success target:

- 5 tasks loaded
- 5 compile pass
- 5 simulation pass
- 5 activity pass
- 5 synthesis pass
- 5 area pass
- 5 fully validated references

If the success target is not met, the report must clearly document the failing task and blocker.

## Optional Reference Metrics

If metrics are deterministic and public-safe, commit only structured files like:

```text
benchmarks/rfid_apbench/tasks/<task_id>/expected/reference_metrics.yaml
```

Do not include logs, VCD snippets, raw tool output, private text, generated RTL, private paths, or absolute work-dir paths.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts
- raw model responses
- generated RTL from model runs
- model outputs
- benchmark run outputs
- VCD files
- simulator logs
- synthesis logs
- compiled simulation artifacts
- tool scratch files
- secrets
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- run model evaluations
- compare models or prompt profiles
- create training data
- create a fine-tuning repository
- claim measured power
- claim final silicon PPA
- integrate private benchmarks

## Definition of Done

Done when:

1. The validation report is regenerated from a tool-enabled run, or a clear environment blocker is documented.
2. All five public/synthetic references are either fully validated or their blockers are clearly reported.
3. Optional reference metrics are committed only if safe, deterministic, and sanitized.
4. `python -m pytest` passes.
5. No raw VCD/log/tool-output/private/model/training artifacts are committed.
6. The report still states this is reference-only validation, not model performance.
