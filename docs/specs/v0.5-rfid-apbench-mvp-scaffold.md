# Feature Spec: v0.5 RFID-APBench MVP Scaffold

## Goal

Turn the v0.5 RFID-APBench planning package into the first executable public/synthetic benchmark scaffold.

This stage should implement the minimum repo structure and utilities needed to score small RFID/NFC-style area/activity tasks later, without running model jobs and without adding private SiliconCraft RTL.

## Inputs

Read first:

- `docs/experiments/v0.5_rfid_apbench_planning.md`
- `reports/v0.5_area_activity_scoring_design.md`
- `docs/specs/rfid_apbench_task_schema.md`
- `docs/specs/fine_tuning_repo_boundary.md`

## Required Scope

Implement only public/synthetic RFID-APBench MVP scaffolding:

1. Benchmark task directory skeleton.
2. Five synthetic task folders with public specs and placeholder/reference fixtures.
3. Adapter/task-loader skeleton.
4. Pure `area_activity` scoring core.
5. VCD toggle-count utility with tests using tiny synthetic VCD fixtures.
6. Sanitized record schema helpers.
7. Unit tests for scoring, schema validation, task loading, and VCD counting.

Do not run model jobs or full benchmark sweeps.

## Required Files

Create or update as appropriate:

- `benchmarks/rfid_apbench/README.md`
- `benchmarks/rfid_apbench/manifest.yaml`
- `benchmarks/rfid_apbench/tasks/ap_001_idle_counter/`
- `benchmarks/rfid_apbench/tasks/ap_002_command_decoder/`
- `benchmarks/rfid_apbench/tasks/ap_003_register_bank_unnecessary_writes/`
- `benchmarks/rfid_apbench/tasks/ap_004_crc_serial_parallel_tradeoff/`
- `benchmarks/rfid_apbench/tasks/ap_005_fsm_controller_idle_activity/`
- `src/rtlbench/adapters/rfid_apbench.py`
- `src/rtlbench/area_activity_scoring.py`
- `src/rtlbench/vcd_activity.py`
- `tests/test_rfid_apbench_adapter.py`
- `tests/test_area_activity_scoring.py`
- `tests/test_vcd_activity.py`

Each task folder should follow the schema from `docs/specs/rfid_apbench_task_schema.md`.

## Task Folder Requirements

Each initial task folder must include public/synthetic files only:

- `task.yaml`
- `prompt.md`
- `reference.sv`
- `testbench.sv`
- `activity_workload.yaml`
- optional `constraints.sdc` only when timing is required
- optional `expected/reference_metrics.yaml` only if metrics are deterministic and sanitized

The task files may be minimal but should be coherent and parseable. They must not contain private RTL, private task text, generated RTL from model runs, logs, VCD files, raw prompts, or model responses.

## Adapter Requirements

The RFID-APBench adapter must:

- load `benchmarks/rfid_apbench/manifest.yaml`
- load each listed task folder
- validate required files and required metadata fields
- expose task id, prompt, reference path, testbench path, top module, activity workload, and timing requirement
- fail clearly on missing task files or invalid metadata
- not require private paths or external private data

Register the adapter only if it fits existing adapter patterns and tests pass. If registration would require broader runner changes, document that as a blocker and keep the adapter importable/tested.

## Area/Activity Scoring Requirements

Implement a pure scoring module for `area_activity`.

It must support:

- gate outcomes
- area metric validation
- activity metric validation
- capped ratios at 2.0
- equal weighting: 0.5 area, 0.5 activity
- invalid score records when gates or metrics fail
- zero-filled aggregate helper for invalid samples
- clear failure categories

Do not use real power fields for activity proxy scoring.

## VCD Activity Requirements

Implement a small VCD toggle-count utility.

It must support:

- counting toggles inside a declared time or cycle window when enough data is available
- excluding selected signals such as clock
- simple total toggle counting
- optional width-weighted counting if easy and tested
- clear errors for missing, malformed, or unsupported VCD input

Do not commit real generated VCD files. Unit tests should use tiny synthetic VCD strings or fixtures that contain no private data.

## Sanitized Output Requirements

Add schema helpers or dataclasses for commit-safe records matching the planning docs.

Records must forbid:

- private RTL
- private task text
- raw prompts
- raw model responses
- generated RTL
- VCD contents
- logs or error logs
- output paths exposing raw artifacts
- secrets
- training datasets
- model weights
- LoRA/QLoRA/DoRA adapters

## Tests

Run:

```bash
python -m pytest
```

Tests must cover:

- adapter loads the five public tasks
- adapter reports missing required files cleanly
- scoring valid area/activity sample
- scoring invalid gate failure
- scoring missing/zero/negative/non-finite area or activity metrics
- aggregate invalid samples contribute zero
- VCD toggle counting basic case
- VCD signal exclusion
- malformed VCD error behavior
- sanitized record rejects forbidden raw/private fields

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts
- raw model responses
- generated RTL from model runs
- model outputs
- benchmark run outputs
- VCD output files
- logs or error logs
- secrets
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- run model evaluations
- run baseline models
- create training data
- create a fine-tuning repository
- claim measured power
- claim final silicon PPA
- integrate private benchmarks
- replace v0.4 RTL-OPT scoring

## Definition of Done

Done when:

1. Five public/synthetic RFID-APBench task folders exist.
2. Adapter/task-loader skeleton loads and validates them.
3. Pure `area_activity` scoring core exists and is tested.
4. VCD toggle-count utility exists and is tested with synthetic fixtures.
5. Sanitized record helpers exist and reject forbidden fields.
6. `python -m pytest` passes.
7. No raw/private/model/training artifacts are committed.
8. README or docs clearly state this is an MVP scaffold, not a completed benchmark result.
