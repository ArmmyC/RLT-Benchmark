# Feature Spec: v0.5 RFID-APBench Reference Validation

## Goal

Validate that the RFID-APBench MVP scaffold is executable on the five public/synthetic reference tasks before any model-generated RTL is evaluated.

This stage is reference-only. It must compile and simulate the committed public reference RTL/testbenches, collect sanitized reference metrics when available, and report any blocker. Do not run model jobs.

## Inputs

Read first:

- `docs/specs/v0.5-rfid-apbench-mvp-scaffold.md`
- `docs/experiments/v0.5_rfid_apbench_planning.md`
- `reports/v0.5_area_activity_scoring_design.md`
- `docs/specs/rfid_apbench_task_schema.md`
- `docs/specs/fine_tuning_repo_boundary.md`

## Required Scope

Implement a reference-validation path for `benchmarks/rfid_apbench` that:

1. Loads the RFID-APBench manifest through the adapter.
2. Validates all five public/synthetic task folders.
3. Compiles and simulates each task's committed `reference.sv` with `testbench.sv` when Icarus Verilog is available.
4. Produces a VCD only in a temporary/work directory, never committed.
5. Counts VCD toggles through `src/rtlbench/vcd_activity.py` using the task workload settings.
6. Runs a generic Yosys synthesis area check when Yosys is available.
7. Writes sanitized reference metrics only when both area and activity are available.
8. Writes a Markdown/CSV validation report.
9. Clearly reports unavailable tools as blockers or skipped metric sources.

## Required Files

Create or update as appropriate:

- `scripts/validate_rfid_apbench_references.py`
- `reports/v0.5_rfid_apbench_reference_validation.md`
- `reports/v0.5_rfid_apbench_reference_validation.csv`
- optional `benchmarks/rfid_apbench/tasks/*/expected/reference_metrics.yaml` only if metrics are deterministic and sanitized
- tests for the validation/report helpers if new helper functions are introduced

## Tool Behavior

The validation script should support:

```bash
python scripts/validate_rfid_apbench_references.py \
  --benchmark-root benchmarks/rfid_apbench \
  --work-dir .tmp/rfid_apbench_reference_validation \
  --output-md reports/v0.5_rfid_apbench_reference_validation.md \
  --output-csv reports/v0.5_rfid_apbench_reference_validation.csv
```

Requirements:

- Use temporary/work directories for compiled simulation artifacts, VCDs, and synthesis scratch files.
- Do not write raw VCDs, logs, compiled artifacts, generated RTL, or tool scratch files into committed paths.
- If `iverilog` is unavailable, report simulation/activity as unavailable and do not fail unclearly.
- If `yosys` is unavailable, report area as unavailable and do not fail unclearly.
- If a tool returns failure, capture only sanitized status and failure category in reports.

## Report Requirements

The Markdown and CSV reports must include per task:

- task_id
- compile status
- simulation status
- VCD/activity status
- synthesis/area status
- timing status
- reference_area when available
- area_unit when available
- reference_activity when available
- activity_metric when available
- failure_category
- notes/blocker

The Markdown report must also include:

- scope and non-goals
- tool availability summary
- aggregate pass counts
- explicit caveat that activity is a VCD toggle-count proxy, not measured power
- explicit caveat that this validates references only, not model performance
- statement that private RTL and fine-tuning remain out of scope

## Optional Reference Metrics

If both reference area and reference activity are deterministic, sanitized, and public-safe, write per-task:

```text
benchmarks/rfid_apbench/tasks/<task_id>/expected/reference_metrics.yaml
```

The YAML must contain only structured metrics:

- schema_version
- task_id
- toolchain_id
- area.value
- area.unit
- activity.value
- activity.metric
- timing.required
- timing.status

Do not include logs, VCD snippets, tool paths, raw outputs, private text, or generated RTL.

## Tests

Run:

```bash
python -m pytest
```

Add or update tests for:

- report row construction from sanitized validation results
- graceful behavior when tools are unavailable, if helper functions are testable
- no raw/private fields in optional reference metrics helper

Do not require real Icarus/Yosys in unit tests unless the existing test suite already does so. Prefer pure helper tests and script-level graceful skip behavior.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts
- raw model responses
- generated RTL from model runs
- model outputs
- benchmark run outputs
- VCD output files
- simulator or synthesis logs
- compiled simulation artifacts
- secrets
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- run model evaluations
- compare models or prompt profiles
- create training data
- create a fine-tuning repository
- claim measured power
- claim final silicon PPA
- integrate private benchmarks

## Definition of Done

Done when:

1. Reference validation script exists.
2. The script validates all five committed public/synthetic tasks.
3. The script handles unavailable Icarus/Yosys gracefully.
4. Sanitized Markdown/CSV reports exist.
5. Optional reference metrics are written only if safe and deterministic.
6. `python -m pytest` passes.
7. No raw VCD/log/tool-output/private/model/training artifacts are committed.
8. The report clearly states this is reference-only validation, not model performance.
