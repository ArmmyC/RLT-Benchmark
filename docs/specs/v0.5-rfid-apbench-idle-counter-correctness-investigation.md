# Feature Spec: v0.5 RFID-APBench Idle Counter Correctness Investigation

## Goal

Investigate the `ap_001_idle_counter` retry outcome after the targeted retry showed that extraction is not the primary blocker anymore: two retry samples extracted, compiled, and synthesized, but both failed public simulation correctness; one sample still failed extraction.

This stage should classify the idle-counter correctness failure using only sanitized committed reports plus bounded local inspection of ignored raw artifacts. It must not commit raw model responses, generated RTL, logs, VCDs, or private data.

This is benchmark analysis only. It is not prompt tuning, model comparison, or fine-tuning.

## Inputs

Read first:

- `reports/v0.5_rfid_apbench_idle_counter_retry.md`
- `reports/v0.5_rfid_apbench_idle_counter_retry.csv`
- `reports/v0.5_rfid_apbench_idle_counter_retry.jsonl`
- `reports/v0.5_rfid_apbench_model_smoke_analysis.md`
- `benchmarks/rfid_apbench/tasks/ap_001_idle_counter/task.yaml`
- `benchmarks/rfid_apbench/tasks/ap_001_idle_counter/prompt.md`
- `benchmarks/rfid_apbench/tasks/ap_001_idle_counter/reference.sv`
- `benchmarks/rfid_apbench/tasks/ap_001_idle_counter/testbench.sv`
- `scripts/run_rfid_apbench_idle_counter_retry.py`
- `scripts/evaluate_rfid_apbench_candidates.py`

## Required Scope

Implement a scoped correctness-investigation pass that:

1. Summarizes the targeted retry outcome from sanitized reports.
2. Classifies the two simulation failures without committing raw generated RTL or simulator logs.
3. Compares the public task spec, reference behavior, and testbench expectations at a high level.
4. Uses ignored local artifacts only to identify high-level failure modes, such as reset behavior, terminal count, clear behavior, idle hold, module interface mismatch, or state-update timing.
5. Produces a sanitized Markdown analysis report and optional CSV summary.
6. Recommends exactly one next action: keep benchmark unchanged, open a prompt-spec clarification, open an extractor bugfix, or run a tiny bounded retry with no prompt changes.
7. Keeps fine-tuning and broad prompt/model experiments out of scope.

## Required Files

Create or update as appropriate:

- `reports/v0.5_rfid_apbench_idle_counter_correctness_investigation.md`
- optional `reports/v0.5_rfid_apbench_idle_counter_correctness_investigation.csv`
- optional pure helper script/tests if needed

Do not commit raw run directories, generated RTL, VCD files, logs, simulator outputs, or model responses.

## Analysis Requirements

The Markdown report must include:

- scope and non-goals
- task id, model, prompt profile, retry sample count
- targeted retry gate summary
- public behavior expectations from the task/testbench
- sanitized per-sample outcome summary
- high-level simulation-failure taxonomy
- likely root cause category for the two simulation failures
- note that one extraction failure still occurred, but correctness is now the stronger blocker
- recommended next action and rationale
- caveat that activity is VCD toggle-count proxy, not measured power
- caveat that this is public synthetic smoke, not private evaluation and not fine-tuning

## Allowed Local Inspection

You may inspect ignored local/output artifacts to classify failures, but committed text must stay high-level and sanitized.

Allowed committed descriptions:

- "failed terminal count expectation"
- "reset behavior mismatch"
- "clear behavior mismatch"
- "idle hold mismatch"
- "interface or top-module mismatch"
- "state update timing mismatch"
- "insufficient evidence from sanitized artifacts"

Forbidden committed content:

- raw generated RTL
- raw model response text
- raw prompts
- simulator log excerpts
- VCD snippets
- absolute output paths exposing artifacts
- private text or private RTL

## Decision Rules

Recommend:

- `keep_benchmark_unchanged` if the public spec, reference, and testbench are internally consistent and failures appear model-caused.
- `prompt_spec_clarification_spec` only if the public prompt/spec is ambiguous in a way that likely caused the failures.
- `extractor_bugfix_spec` only if the extracted RTL was correct but extraction damaged or truncated it.
- `tiny_retry_no_prompt_change` only if the evidence is inconclusive and no task/testbench issue is found.

Do not recommend fine-tuning from this one task.

## Tests

Run:

```bash
python -m pytest
```

Add tests only if helper code is introduced. Do not require live model endpoints in unit tests.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts
- raw model responses
- generated RTL
- model outputs
- raw benchmark run outputs
- VCD files
- simulator logs
- synthesis logs
- compiled simulation artifacts
- tool scratch files
- secrets or endpoint credentials
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- modify task text, prompt, reference, testbench, extractor, or evaluator unless a separate follow-up spec is created
- run more model samples
- compare multiple models
- tune prompts broadly
- create training data
- fine-tune a model
- claim measured power
- claim final silicon PPA
- integrate private benchmarks

## Definition of Done

Done when:

1. A sanitized correctness-investigation Markdown report exists.
2. The two simulation failures are classified at a high level without raw artifacts.
3. The report states whether the benchmark should remain unchanged or needs a scoped follow-up spec.
4. `python -m pytest` passes.
5. No raw/private/model-output/training artifacts are committed.
