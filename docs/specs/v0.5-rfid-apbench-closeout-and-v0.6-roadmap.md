# Feature Spec: v0.5 RFID-APBench Closeout and v0.6 Roadmap

## Goal

Close out v0.5 RFID-APBench as a public/synthetic area plus activity benchmark MVP and define the bounded v0.6 roadmap.

RFID-APBench remains a benchmark and evaluation effort. It is not a private RTL benchmark, not a training-data pipeline, and not a fine-tuning project.

## v0.5 Closeout State

v0.5 established an executable public/synthetic RFID/NFC-style benchmark flow with five tasks, reference validation, candidate-fixture evaluation, one bounded model smoke, idle-counter retry analysis, and idle-counter correctness investigation.

The closeout evidence is:

- Reference validation: 5/5 references compile, simulate, produce activity metrics, synthesize, and expose area metrics.
- Candidate fixture smoke: 5/5 `reference_copy` candidates pass compile, correctness, synthesis, and area/activity scoring with mean score 1.000000.
- First real model smoke: `qwen36-27b`, 5 tasks, 1 sample each, `neutral_baseline`, 4/5 extracted and evaluated with valid scores.
- First real model smoke scores: 4 valid scores, mean valid score 0.946946, all-sample zero-filled mean 0.757557.
- Idle-counter targeted retry: exactly 3 samples for `ap_001_idle_counter`; 2/3 extracted, compiled, and synthesized, 0/3 passed correctness, 0 valid scores.
- Idle-counter correctness investigation: the two simulation failures are classified as model-produced state-update timing mismatch / terminal-count expectation failure.
- Closeout pytest status: 223 passed, 1 skipped.

## Benchmark Asset Decision

Benchmark assets remain unchanged at v0.5 closeout.

Do not modify:

- task text
- prompts
- references
- testbenches
- extractor
- evaluator
- model-smoke scripts

Rationale:

- The five public references validate end-to-end.
- The public `reference_copy` candidate fixture validates end-to-end.
- The idle-counter prompt, reference, and testbench are internally consistent.
- The idle-counter model retry failures reached public simulation and failed a specific behavioral expectation, which indicates model output behavior rather than a benchmark asset defect.
- There is no current evidence for an extractor bug, evaluator bug, task ambiguity, or prompt/spec defect that justifies mutating v0.5 assets.

## Fine-Tuning Boundary

Fine-tuning is not ready.

Do not add:

- private RTL
- private task text
- raw prompts
- raw model responses
- generated RTL
- raw model outputs
- VCDs, simulator logs, synthesis logs, compiled artifacts, or tool scratch outputs
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

Rationale:

- v0.5 has only five public/synthetic tasks.
- The first model smoke used one sample per task.
- The only targeted retry covered one task and produced no valid idle-counter score.
- The benchmark still needs repeated-sample behavior and broader task coverage before it can serve as a stable evaluation dependency for any future training repository.

## v0.6 Roadmap

v0.6 should remain a public/synthetic benchmark-evaluation release. It should not introduce private RTL, training data, or fine-tuning.

### 1. Three-Sample Baseline On Existing Five Tasks

Run the existing five public/synthetic RFID-APBench tasks with the same configured/default model and `neutral_baseline` prompt profile.

Required bounds:

- 5 existing tasks
- 3 samples per task
- same benchmark assets
- same evaluator/scoring path
- low/deterministic generation settings where supported
- raw outputs only in ignored output/work directories
- committed artifacts must be sanitized Markdown/CSV/JSONL summaries

Purpose:

- distinguish one-off extraction issues from repeatable task fragility
- measure per-task score variance
- produce a more honest baseline than 1 sample per task

### 2. Expand From 5 To 10-15 Public Synthetic RFID/AP Tasks

Add 5 to 10 new public/synthetic RFID/NFC-style area plus activity tasks.

Task expansion should cover:

- counters and timers
- command/control decoders
- small register banks
- CRC or parity datapaths
- frame/control FSMs
- low-activity idle behavior
- clear/reset/hold edge cases

Every new task must include:

- public synthetic prompt
- public reference RTL
- public testbench
- declared VCD activity workload
- reference area metric
- reference activity metric
- sanitized task metadata

No private RTL or private task text may be used.

### 3. Failure Taxonomy Report

Create a v0.6 failure taxonomy report across the 3-sample baseline.

Minimum categories:

- endpoint or request failure
- empty response
- incomplete RTL
- extraction failure
- interface or top-module mismatch
- compile failure
- reset behavior mismatch
- clear behavior mismatch
- terminal count mismatch
- idle hold mismatch
- state-update timing mismatch
- simulation correctness failure
- synthesis failure
- area metric unavailable
- activity metric unavailable
- score regression

The taxonomy must remain sanitized and must not include raw generated RTL, raw model responses, raw prompts, logs, VCD snippets, or artifact paths exposing raw outputs.

### 4. Sanitized Model-Evaluation Records

Define and emit stable sanitized model-evaluation records for RFID-APBench.

Records should include:

- benchmark
- task id
- sample id
- model
- prompt profile
- generation settings
- generation status
- extraction status
- compile status
- correctness status
- synthesis status
- timing status if required
- area metric status and value
- activity metric status and value
- area score
- activity score
- final score
- failure category
- sanitized notes

Records must exclude:

- raw prompts
- raw responses
- generated RTL
- raw logs
- VCD contents
- output paths to raw artifacts
- secrets
- private data
- training artifacts

### 5. No Fine-Tuning Yet

v0.6 must not start fine-tuning.

Fine-tuning remains blocked until RFID-APBench has:

- 10-15 stable public/synthetic tasks
- repeated-sample baseline results
- stable sanitized evaluation records
- a useful failure taxonomy
- clear benchmark/repo boundary from any future training repository
- no private or raw model artifacts in committed history

## Definition Of Done For v0.6

v0.6 is complete when:

- existing five tasks have a 3-sample sanitized model baseline
- RFID-APBench has 10-15 public/synthetic tasks or a committed expansion report explaining any shortfall
- reference validation passes for all committed tasks
- candidate fixture smoke passes for all committed tasks
- model-evaluation records are sanitized and stable
- failure taxonomy report exists
- pytest passes
- no raw/private/model/training artifacts are committed
