# Feature Spec: v0.7 RFID-APBench Prompt-Profile Experiment Rejected Closeout

## Goal

Close the bounded v0.7 prompt-profile experiment and record that the tested `completion_reliability` challenger profile did not improve the targeted empty-response behavior.

The bounded experiment recommends `profile_rejected_closeout_spec` because the challenger matched the control on empty-response rate and downstream gates:

- `neutral_baseline`: 5/6 `success_empty`, 0/6 valid scores
- `completion_reliability`: 5/6 `success_empty`, 0/6 valid scores
- both profiles had identical empty rates by task
- no benchmark assets should change

This closeout is documentation and planning only. It is not another model run, prompt tuning, benchmark modification, model comparison, private evaluation, fine-tuning, or training-data creation.

## Inputs

Read first:

- `reports/v0.7_rfid_apbench_bounded_prompt_profile_experiment.md`
- `reports/v0.7_rfid_apbench_bounded_prompt_profile_experiment.csv`
- `reports/v0.7_rfid_apbench_bounded_prompt_profile_experiment.jsonl`
- `reports/v0.7_rfid_apbench_targeted_empty_response_observability_rerun.md`
- `reports/v0.7_rfid_apbench_runner_endpoint_observability.md`
- `reports/v0.7_rfid_apbench_model_output_robustness.md`
- `configs/prompt_profiles.yaml`

## Required Scope

Create a sanitized closeout report that:

1. Summarizes the bounded prompt-profile experiment scope.
2. States the exact profiles compared.
3. Summarizes request outcomes and empty-response rates.
4. Summarizes gate outcomes and valid-score counts.
5. States that the `completion_reliability` challenger is rejected for further validation.
6. States that benchmark task prompts/assets remain unchanged.
7. States that no additional prompt-profile experiment is justified from this evidence alone.
8. States fine-tuning remains out of scope and not recommended.
9. Recommends exactly one next bounded direction.
10. Suggests whether to keep, remove, or leave unused the `completion_reliability` profile.

## Decision Rules

Recommend exactly one:

- `v0.7_empty_response_findings_closeout_spec` if the best next step is to close the empty-response investigation thread and summarize findings.
- `endpoint_runtime_investigation_spec` only if the closeout finds the repeated token-limit null-content behavior needs environment/provider-side investigation before more benchmark work.
- `no_immediate_followup_beyond_documentation` if no further v0.7 work is justified now.

Do not recommend additional prompt-profile experiments unless there is new evidence not present in the bounded experiment report.

Do not recommend fine-tuning, private evaluation, model comparison, benchmark modification, or training-data creation from this closeout.
