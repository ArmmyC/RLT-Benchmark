# Feature Spec: v0.7 RFID-APBench Bounded Thinking-Disabled Validation

## Goal

Validate whether forwarding `chat_template_kwargs.enable_thinking=false` into RFID-APBench requests changes the previously observed null-content token-limit behavior on the targeted public/synthetic tasks.

The thinking-mode propagation bugfix is complete: future `qwen36-27b` RFID generation calls now carry the configured model preset `extra_body`. This validation should confirm the post-bugfix behavior using a very small, sanitized run before considering max-token changes, prompt-profile work, broad reruns, or any training-related work.

This is a bounded validation step only. It is not benchmark modification, prompt tuning, model comparison, private evaluation, fine-tuning, or training-data creation.

## Inputs

Read first:

- `reports/v0.7_rfid_apbench_thinking_mode_configuration_bugfix.md`
- `reports/v0.7_rfid_apbench_endpoint_runtime_investigation.md`
- `reports/v0.7_rfid_apbench_profile_rejected_closeout.md`
- `reports/v0.7_rfid_apbench_bounded_prompt_profile_experiment.md`
- `reports/v0.7_rfid_apbench_targeted_empty_response_observability_rerun.md`
- `configs/models.yaml`
- `configs/prompt_profiles.yaml`
- `scripts/run_rfid_apbench_3sample_baseline.py`
- `src/rtlbench/client.py`

## Required Scope

Run a tiny post-bugfix validation limited to:

- tasks: `ap_001_idle_counter`, `ap_010_retry_timeout_fsm`
- samples: at most 1 sample per task
- profile: `neutral_baseline`
- model: configured/default `qwen36-27b` when available
- max tokens: unchanged `4096`
- temperature/top-p: unchanged deterministic defaults
- benchmark assets: unchanged
- extractor/evaluator/scoring: unchanged
- committed outputs: sanitized reports only

The validation must use the fixed RFID runner path so requests include the configured model preset `extra_body`, especially `chat_template_kwargs.enable_thinking=false`.

If endpoint or tools are unavailable, do not fake results. Emit sanitized blocker/failure rows or a sanitized note.

## Required Files

Create:

- `reports/v0.7_rfid_apbench_bounded_thinking_disabled_validation.md`

Optional if a live validation is performed:

- `reports/v0.7_rfid_apbench_bounded_thinking_disabled_validation.csv`
- `reports/v0.7_rfid_apbench_bounded_thinking_disabled_validation.jsonl`

Do not overwrite historical reports.

## Report Requirements

The Markdown report must include:

- scope and non-goals
- source evidence used
- validation configuration
- confirmation that the fixed path includes the thinking-disable extra body
- endpoint and tool status without secrets
- total attempted samples
- per-sample sanitized request outcome summary
- content presence, character count, finish reason, completion-token count, parse status, and HTTP class
- extraction/evaluation gate summary if candidates are produced
- comparison against the pre-bugfix null-content token-limit pattern
- interpretation of whether the symptom changed, persisted, or could not be tested
- benchmark asset decision
- fine-tuning readiness decision
- exactly one recommended next step
- activity proxy caveat
- public synthetic / no private / not fine-tuning caveat

## Decision Rules

Recommend exactly one:

- `targeted_3sample_thinking_disabled_validation_spec` if the one-sample validation shows the null-content token-limit symptom improves and a slightly larger targeted confirmation is justified.
- `bounded_max_tokens_diagnostic_spec` if null-content token-limit behavior persists even with thinking disabled and max-token budget becomes the next bounded diagnostic.
- `serving_stack_issue_spec` if null-content behavior persists and evidence points to endpoint/server behavior rather than token budget or RTLBench configuration.
- `client_response_capture_observability_spec` if sanitized metadata is still insufficient to interpret the response boundary.
- `no_immediate_followup_beyond_documentation` if validation cannot be performed or yields no useful evidence.

Do not recommend fine-tuning, private evaluation, model comparison, broad benchmark rerun, prompt-profile experiment, benchmark modification, or training-data creation.


## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts beyond public benchmark prompts/profile text
- raw model responses
- response bodies
- generated model RTL
- model outputs outside sanitized reports
- raw benchmark output directories
- VCD files
- simulator logs
- synthesis logs
- compiled simulation artifacts
- tool scratch files
- secrets or endpoint credentials
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- add or modify benchmark tasks
- edit benchmark task prompt files
- run a benchmark-wide model evaluation
- compare different models
- change prompt profiles
- raise the global max-token default
- create training data
- fine-tune a model
- add private benchmarks
- claim measured power
- claim final silicon PPA
- change area/activity scoring formula
- change evaluator policy

## Definition of Done

Done when:

1. `reports/v0.7_rfid_apbench_bounded_thinking_disabled_validation.md` exists.
2. It validates at most one sample each for `ap_001_idle_counter` and `ap_010_retry_timeout_fsm`, or explicitly reports endpoint/tool blockers.
3. It confirms the fixed runner path includes the thinking-disable extra body.
4. It reports sanitized request/content/token metadata without raw outputs.
5. It states whether the null-content token-limit symptom changed, persisted, or could not be tested.
6. Benchmark assets remain unchanged.
7. Fine-tuning is not recommended from this evidence alone.
8. Exactly one next bounded step is recommended.
9. `python -m pytest` passes.
10. No raw/private/model-output/training artifacts are committed.
