# Feature Spec: v0.3 qwen36-27b Prompt Experiment Expansion

## 1. Goal

Expand Baseline v0.3 prompt experiments from VerilogEval smoke into a controlled, staged prompt-profile evaluation for `qwen36-27b`.

The VerilogEval smoke stage is complete. All four prompt profiles finished with 3/3 passing samples:

- `neutral_baseline`: 3/3 passed
- `strict_rtl_only`: 3/3 passed
- `low_area_low_power`: 3/3 passed
- `no_explanation_code_only`: 3/3 passed

The next goal is to run RTLLM smoke for the same four prompt profiles before any RTL-OPT smoke or full benchmark run.

## 2. Research Question

Can prompt profile changes improve RTL generation behavior for `qwen36-27b` while keeping model, benchmark, temperature, top-p, max tokens, evaluator, and task count controlled?

This is a prompt experiment, not a model comparison.

## 3. Prompt Profiles

The active v0.3 profiles are:

1. `neutral_baseline`
2. `strict_rtl_only`
3. `low_area_low_power`
4. `no_explanation_code_only`

All four profiles should be tested on every smoke benchmark before selecting larger-run candidates.

## 4. Staged Expansion Plan

### Stage A: Complete VerilogEval smoke

Status: complete.

All four VerilogEval prompt profiles are finished and recorded in:

- `reports/v0.3_qwen36_27b_prompt_smoke_summary.md`
- `reports/v0.3_qwen36_27b_prompt_smoke_summary.csv`

### Stage B: Review VerilogEval smoke

Status: passed.

Review result:

- all four profiles are `finished`
- each has 3 samples
- all four pass 3/3
- no raw outputs were committed
- result is safe to expand to RTLLM smoke

### Stage C: RTLLM smoke

Status: next.

Run RTLLM smoke for all four profiles:

- `neutral_baseline`
- `strict_rtl_only`
- `low_area_low_power`
- `no_explanation_code_only`

Then regenerate the smoke summary so it includes VerilogEval and RTLLM rows together.

### Stage D: Review RTLLM smoke

Before expanding to RTL-OPT, check:

- all four RTLLM profiles are `finished`
- each has 3 samples
- summary counts are plausible
- no missing run metadata
- no raw outputs are committed
- prompt profile is recorded in metadata
- the report clearly shows VerilogEval and RTLLM results

### Stage E: RTL-OPT equivalence smoke

Only if Stage D looks clean, run RTL-OPT equivalence smoke for all four profiles.

### Stage F: Decide full-run candidate

Only after all smoke stages pass, choose one or two profiles for larger runs. Do not automatically run full benchmarks.

## 5. Required Reports

The existing smoke summary report should remain the single human-readable report for v0.3 smoke:

- `reports/v0.3_qwen36_27b_prompt_smoke_summary.md`
- `reports/v0.3_qwen36_27b_prompt_smoke_summary.csv`

After Stage C, the reports should include:

- all four VerilogEval rows
- all four RTLLM rows

Missing combinations requested by the summarizer should remain explicitly marked as missing. Do not fabricate missing results.

## 6. Commit Rules

Commit only safe reports and source/docs changes.

Allowed:

- `reports/v0.3_qwen36_27b_prompt_smoke_summary.md`
- `reports/v0.3_qwen36_27b_prompt_smoke_summary.csv`
- documentation/spec updates
- tests or helper fixes if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 7. Non-goals

Do not:

- run all full benchmarks
- run RTL-OPT in the RTLLM stage
- add new models
- change benchmark logic
- change evaluator behavior
- modify Baseline v0.1 or v0.2 artifacts
- implement LANTA serving, SSH, Slurm swapping, OpenWebUI, or LiteLLM logic
- claim PPA improvements from smoke results alone

## 8. Definition of Done for Next Codex Task

The next Codex task is done when:

1. RTLLM smoke has finished for all four prompt profiles.
2. The v0.3 smoke summary report includes all four VerilogEval profiles and all four RTLLM profiles.
3. The summary CSV includes all four VerilogEval profiles and all four RTLLM profiles.
4. `python -m pytest` passes.
5. No raw experiment outputs are committed.
6. The result summary states whether it is safe to move to RTL-OPT equivalence smoke.
