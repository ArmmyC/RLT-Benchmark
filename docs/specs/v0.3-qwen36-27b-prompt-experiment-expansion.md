# Feature Spec: v0.3 qwen36-27b Prompt Experiment Closeout

## 1. Goal

Close out the v0.3 `qwen36-27b` prompt-profile experiment as a documented, reproducible experiment package.

All active experiment stages are complete:

1. prompt-profile implementation
2. smoke scaffolding
3. VerilogEval smoke
4. RTLLM smoke
5. RTL-OPT equivalence smoke
6. candidate decision
7. larger-validation scaffold
8. larger-validation execution
9. larger-validation decision

The final decision is conservative:

- keep `neutral_baseline` as the default prompt profile for now
- keep `strict_rtl_only` as an optional challenger
- do not promote `low_area_low_power` yet
- do not promote `no_explanation_code_only`

The next step is not another benchmark run. The next step is closeout documentation and an index of the v0.3 artifacts.

## 2. Completed Results

### Smoke results

- `neutral_baseline`: 7/9
- `strict_rtl_only`: 7/9
- `low_area_low_power`: 6/9
- `no_explanation_code_only`: 5/9

### Larger validation results

- `neutral_baseline`: 31/40
- `strict_rtl_only`: 31/40

Interpretation:

- `strict_rtl_only` was slightly better on VerilogEval larger validation.
- `neutral_baseline` was slightly better on RTL-OPT equivalence larger validation.
- RTLLM larger validation was tied.
- There is no clear evidence to switch the default prompt profile.

## 6. Commit Rules

Allowed:

- closeout report
- README or docs updates that point to the closeout report
- documentation/spec updates only if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 7. Non-goals

Do not:

- run more benchmarks
- run full benchmarks
- add new models
- change benchmark logic
- change evaluator behavior
- modify Baseline v0.1 or v0.2 artifacts
- create or push git tags
- implement LANTA serving, SSH, Slurm swapping, OpenWebUI, or LiteLLM logic
- claim final silicon PPA quality

## 8. Definition of Done for Next Codex Task

The next Codex task is done when:

1. Closeout Markdown report exists.
2. Closeout report links or lists all v0.3 report artifacts.
3. Closeout report records the final recommendation.
4. README or docs point to the closeout report.
5. Suggested tag name is documented, but no tag is created.
6. `python -m pytest` passes.
7. No raw experiment outputs are committed.
