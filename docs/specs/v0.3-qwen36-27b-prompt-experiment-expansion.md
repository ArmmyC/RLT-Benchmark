Add or update scripts to support larger validation:

- dry-run commands for the selected benchmark/profile/limit matrix

- a summary script or summary mode for larger validation outputs

Suggested report paths for later summaries:

- `reports/v0.3_qwen36_27b_prompt_larger_summary.md`

## 6. Commit Rules

Allowed:

- scripts/helpers for larger validation
- tests
- documentation/spec updates
- README usage examples

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 7. Non-goals

Do not:

- execute the larger validation slice yet
- run full benchmarks
- add new models
- change benchmark logic
- change evaluator behavior
- modify Baseline v0.1 or v0.2 artifacts
- implement LANTA serving, SSH, Slurm swapping, OpenWebUI, or LiteLLM logic
- claim final prompt superiority from smoke results

## 8. Definition of Done for Next Codex Task

The next Codex task is done when:

1. Larger-validation command scaffolding exists and defaults to dry-run.
2. The scaffold targets only `neutral_baseline` and `strict_rtl_only`.
3. The scaffold uses separate `v0.3_prompt_profiles_larger` output roots.
4. The planned limits are VerilogEval 20, RTLLM 10, RTL-OPT 10.
5. A larger-summary reader exists or existing summary code supports the larger output root.
6. README/docs explain how to dry-run, execute later, and summarize.
7. `python -m pytest` passes.
8. No model runs or raw outputs are committed.
