# Feature Spec: v0.3 qwen36-27b Prompt Experiment Expansion

## 1. Goal

Expand Baseline v0.3 prompt experiments from VerilogEval and RTLLM smoke into a controlled, staged prompt-profile evaluation for `qwen36-27b`.

The VerilogEval smoke stage is complete. All four prompt profiles finished with 3/3 passing samples:

- `neutral_baseline`: 3/3 passed
- `strict_rtl_only`: 3/3 passed
- `low_area_low_power`: 3/3 passed
- `no_explanation_code_only`: 3/3 passed

The RTLLM smoke stage is complete. The first three prompt profiles tied at 2/3 passing samples, while `no_explanation_code_only` was weaker at 1/3:

- `neutral_baseline`: 2/3 passed
- `strict_rtl_only`: 2/3 passed
- `low_area_low_power`: 2/3 passed
- `no_explanation_code_only`: 1/3 passed

The next goal is to run RTL-OPT equivalence smoke for the same four prompt profiles before selecting any larger-run candidates.

## 2. Research Question

Can prompt profile changes improve RTL generation behavior for `qwen36-27b` while keeping model, benchmark, temperature, top-p, max tokens, evaluator, and task count controlled?

This is a prompt experiment, not a model comparison.

## 3. Prompt Profiles

The active v0.3 profiles are:

1. `neutral_baseline`
2. `strict_rtl_only`
3. `low_area_low_power`
4. `no_explanation_code_only`

All four profiles should be tested on every smoke benchmark before selecting larger-run candidates. Based on RTLLM smoke, `no_explanation_code_only` should be treated as a riskier candidate unless RTL-OPT gives a strong reason to keep it.

## 6. Commit Rules

Commit only safe reports and source/docs changes.

Allowed:

- `reports/v0.3_qwen36_27b_prompt_smoke_summary.md`
- `reports/v0.3_qwen36_27b_prompt_smoke_summary.csv`
- documentation/spec updates
- tests or helper fixes if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 7. Non-goals

Do not:

- run all full benchmarks
- run full RTL-OPT
- add new models
- change benchmark logic
- change evaluator behavior
- modify Baseline v0.1 or v0.2 artifacts
- implement LANTA serving, SSH, Slurm swapping, OpenWebUI, or LiteLLM logic
- claim PPA improvements from smoke results alone

4. `python -m pytest` passes.
