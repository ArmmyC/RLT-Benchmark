# Feature Spec: v0.6 RFID-APBench Public Task Expansion

## Goal

Expand RFID-APBench from 5 to 10 public/synthetic RFID/AP-style tasks after the post-tool-health 3-sample baseline showed the current benchmark assets are stable enough for the next benchmark iteration.

This expansion should improve benchmark breadth while preserving the validated tool-health gates, sanitized reporting, and existing area/activity scoring contract.

This is benchmark expansion only. It is not model comparison, prompt tuning, fine-tuning, private evaluation, or training-data creation.

## Required Scope

Add exactly five new public/synthetic RFID/AP-style tasks so the benchmark reaches 10 total tasks.

For each new task, create:

- `benchmarks/rfid_apbench/tasks/<task_id>/task.yaml`
- `benchmarks/rfid_apbench/tasks/<task_id>/prompt.md`
- `benchmarks/rfid_apbench/tasks/<task_id>/reference.sv`
- `benchmarks/rfid_apbench/tasks/<task_id>/testbench.sv`
- `benchmarks/rfid_apbench/tasks/<task_id>/constraints.sdc`
- `benchmarks/rfid_apbench/tasks/<task_id>/activity_workload.yaml`
- `benchmarks/rfid_apbench/tasks/<task_id>/expected/reference_metrics.yaml` after validation

Also update:

- `benchmarks/rfid_apbench/manifest.yaml`
- `benchmarks/rfid_apbench/README.md`
- `benchmarks/rfid_apbench/candidates/reference_copy/<task_id>.sv`
- sanitized validation/smoke reports

## New Task Themes

Use public/synthetic tasks only. Suggested themes:

1. low-duty-cycle edge detector or wakeup pulse filter
2. small command-frame length/check parser
3. register write-enable gating with byte lanes
4. serial parity/checksum accumulator with clear/hold behavior
5. small low-activity FSM with timeout or retry counter

Each task should be RFID/AP-relevant, small, synthesizable, and deterministic. Avoid copying proprietary SiliconCraft behavior or private RTL.

## Task Requirements

Each new task must:

- have a unique stable task id, preferably `ap_006_*` through `ap_010_*`
- define one required top module
- use public synthetic behavior only
- include a concise prompt suitable for model generation
- include a synthesizable SystemVerilog reference
- include a self-checking testbench
- include an activity workload with a measurement window and signal exclusions if needed
- include constraints file, even if timing is not required
- pass reference validation under healthy Icarus, `vvp`, and Yosys
- pass `reference_copy` candidate fixture smoke
- produce reference area and VCD activity metrics

## Required Reports

Create or update sanitized reports:

- `reports/v0.6_rfid_apbench_task_expansion_reference_validation.md`
- `reports/v0.6_rfid_apbench_task_expansion_reference_validation.csv`
- `reports/v0.6_rfid_apbench_task_expansion_candidate_smoke.md`
- `reports/v0.6_rfid_apbench_task_expansion_candidate_smoke.csv`
- optional JSONL if consistent with current tooling

Reports must include:

- scope and non-goals
- total task count after expansion
- tool health status
- per-task compile/simulation/activity/synthesis/area status
- per-task reference metrics
- candidate fixture pass/fail summary
- activity proxy caveat
- public synthetic / not private / not fine-tuning caveat

## Tool and Validation Requirements

Before accepting the expansion:

1. Tool health checks for Icarus, `vvp`, and Yosys must be healthy.
2. Reference validation must pass for all 10 tasks.
3. `reference_copy` candidate fixture evaluation must pass for all 10 tasks.
4. Generated reports must stay sanitized.
5. `python -m pytest` must pass.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- private protocols or confidential behavior
- raw prompts beyond public benchmark prompts
- raw model responses
- generated model RTL
- model outputs
- raw benchmark output directories
- VCD files
- simulator logs
- synthesis logs
- compiled simulation artifacts
- tool scratch files
- secrets or endpoint credentials
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- run model baseline on the expanded task set yet
- compare models
- tune prompts
- create training data
- fine-tune a model
- add private benchmarks
- claim measured power
- claim final silicon PPA
- change area/activity scoring formula
- change evaluator policy except for clear validation bugs found during task expansion

## Definition of Done

Done when:

1. RFID-APBench contains exactly 10 public/synthetic tasks.
2. All new task files are present and included in the manifest.
3. All 10 references pass compile, simulation, activity, synthesis, and area validation.
4. `reference_copy` candidate fixture passes for all 10 tasks.
5. Sanitized validation and candidate-smoke reports are committed.
6. `python -m pytest` passes.
7. No raw/private/model-output/training artifacts are committed.
