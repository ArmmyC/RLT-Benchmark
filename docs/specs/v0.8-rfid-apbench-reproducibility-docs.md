# Feature Spec: v0.8 RFID-APBench Reproducibility Docs

## Goal

Create user-facing reproducibility documentation for the v0.7 RFID-APBench post-fix baseline.

The v0.8 release checklist recommends `v0.8_reproducibility_docs_spec` because the release checklist now provides a pre-tag gate, and the next useful hardening step is to document how to reproduce the v0.7 post-fix baseline with environment setup, deterministic settings, command shape, output locations, and artifact boundaries.

This is documentation hardening only. It is not a model evaluation, benchmark modification, prompt-profile experiment, model comparison, private evaluation, fine-tuning, or training-data creation.

## Required Scope

Create reproducibility documentation that explains how to reproduce the v0.7 RFID-APBench post-fix baseline at a high level without exposing secrets or requiring private infrastructure.

The docs must cover:

1. Public/synthetic benchmark scope and current 10-task manifest.
2. Required local tools: Python environment, Icarus Verilog/`vvp`, and Yosys.
3. Endpoint environment configuration: `QWEN_BASE_URL`, `QWEN_API_KEY`, optional `QWEN_MODEL`, optional `QWEN_TIMEOUT`, and `.env` handling.
4. Deterministic generation settings: `qwen36-27b`, `neutral_baseline`, temperature `0.0`, top-p `1.0`, max tokens `4096`, 3 samples per task.
5. Model preset behavior: `chat_template_kwargs.enable_thinking=false` is forwarded from `configs/models.yaml` for `qwen36-27b`.
6. Command shape for a full 10-task × 3-sample run and optional targeted validations.
7. Expected sanitized output files: Markdown, CSV, JSONL reports under `reports/`.
8. Ignored raw artifact boundaries: raw responses, generated/extracted RTL, VCDs, logs, compiled artifacts, scratch/output dirs.
9. How to verify tool health, report contents, and no-secret/no-raw-artifact policy before committing.
10. Known v0.7 limitation: `ap_006_wakeup_edge_filter` candidate limitation.
11. What the benchmark does and does not prove: VCD toggle-count activity proxy, not measured power; public/synthetic, not private evaluation; evaluation artifact, not fine-tuning readiness.

## Report Requirements

The Markdown report must include:

- scope and non-goals
- source evidence used
- summary of reproducibility document created
- sections covered in the reproducibility docs
- how users should use the docs with the release checklist
- benchmark asset decision
- fine-tuning readiness decision
- exactly one recommended next bounded direction
- activity proxy caveat
- public synthetic / no private / not fine-tuning caveat

## Reproducibility Doc Requirements

`docs/release/rfid_apbench_reproducibility.md` must be practical and user-facing. It should include:

- prerequisite checklist
- environment variable examples with placeholder values only
- `.env` example without real secrets
- command examples using existing scripts
- explanation of sanitized report outputs
- explanation of ignored/raw artifact outputs
- verification checklist before committing results
- troubleshooting notes for endpoint unavailable, tool unavailable, request failed, extraction failure, compile/simulation/synthesis failure, and empty/null-content recurrence
- caveats and non-goals

## Decision Rules

Recommend exactly one:

- `v0.8_report_hygiene_spec` if reproducibility docs are complete and report schema/lineage cleanup should happen next.
- `v0.8_ci_release_assertions_spec` if automated release/reproducibility checks should happen next.
- `no_immediate_followup_beyond_documentation` if the reproducibility docs are sufficient for now.

Do not recommend fine-tuning, private evaluation, model comparison, global max-token changes, prompt-profile experiments, benchmark modification, broad rerun, or training-data creation from this documentation step.

## Tests

Run:

```bash
python -m pytest
```

Add tests only if helper code is introduced. Do not require a live model endpoint in unit tests.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts beyond public benchmark prompt text
- raw model responses
- response bodies
- generated model RTL
- model outputs outside sanitized reports
- raw benchmark output directories
- VCD files
- simulator logs
- synthesis logs
- compiled artifacts
- tool scratch files
- secrets or endpoint credentials
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- run another model evaluation
- add or modify benchmark tasks
- edit prompt text
- edit reference RTL or testbench behavior
- change module interfaces
- compare models
- change prompt profiles
- raise global max-token default
- create training data
- fine-tune a model
- add private benchmarks
- claim measured power
- claim final silicon PPA
- change workload, metrics, area/activity scoring, or evaluator policy
- create or push git tags

## Definition of Done

Done when:

1. `docs/release/rfid_apbench_reproducibility.md` exists.
2. `reports/v0.8_rfid_apbench_reproducibility_docs.md` exists.
3. The docs explain environment setup, deterministic settings, command shape, outputs, raw artifact boundaries, verification, troubleshooting, and caveats.
4. The report summarizes how the docs should be used with the release checklist.
5. Benchmark assets remain unchanged.
6. Fine-tuning is not recommended from this evidence alone.
7. Exactly one next bounded direction is recommended.
8. `python -m pytest` passes.
9. No raw/private/model-output/training artifacts are committed.
