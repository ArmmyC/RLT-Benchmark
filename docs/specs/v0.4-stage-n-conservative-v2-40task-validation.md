# Feature Spec: v0.4 Stage N Conservative Area v2 40-Task Validation

## 1. Goal

Run a larger but still bounded public RTL-OPT validation for `conservative_area_v2` after Stage M showed it is a promising challenger to `neutral_baseline`.

Stage M showed:

- `neutral_baseline`: 14 valid PPA records out of 20
- `conservative_area_v2`: 17 valid PPA records out of 20
- paired-valid tasks: 14
- v2 improved: 1
- v2 matched: 12
- v2 regressed: 1
- v2 gained validity on `addr_calcu`, `calculation`, and `divider_8bit`
- v2 lost validity on 0 tasks
- v2 still had a paired score regression on `adder`

The next step is not full RTL-OPT and not private SiliconCraft tasks. Run one larger public validation to decide whether `conservative_area_v2` should remain the challenger, be revised again, or be rejected.

This remains a public-flow generic-cell proxy experiment. It is not final silicon PPA and has no measured power data.

## 2. Next Stage

### Stage N: 40-task matched `neutral_baseline` vs `conservative_area_v2` validation

Status: next.

Compare only:

- `neutral_baseline`
- `conservative_area_v2`

Do not run `conservative_area`, `low_area_low_power`, or `strict_rtl_only`.

## 3. Required Scope

Run only:

- benchmark: `rtlopt`
- model: `qwen36-27b`
- prompt profiles: `neutral_baseline`, `conservative_area_v2`
- task count: 40 tasks
- samples per task: 1
- evaluator: `yosys_equiv`
- scoring: v0.4 sanitized PPA export, `area_only`, generic-cell proxy when mapped area is unavailable

Task selection:

- Use a deterministic 40-task public RTL-OPT slice.
- Include all 20 Stage L/M tasks when possible.
- Record the exact task list in the report.
- Do not compare unmatched tasks.

## 4. Required Preflight

Before model calls:

1. Run `python scripts/check_yosys_techmap.py --yosys yosys`.
2. If preflight fails, stop and do not run model jobs.
3. Confirm the task list and output location.

## 5. Required Artifacts

Create sanitized artifacts:

- `artifacts/v0.4/ppa_validation40/rtlopt_neutral_baseline_ppa.jsonl`
- `artifacts/v0.4/ppa_validation40/rtlopt_conservative_area_v2_ppa.jsonl`

Create reports:

- `reports/v0.4_ppa_validation40_neutral_summary.csv`

- `reports/v0.4_ppa_validation40_neutral_vs_conservative_v2.md`
- `reports/v0.4_ppa_validation40_neutral_vs_conservative_v2.csv`

## 6. Comparison Rules

The matched comparison report must include:

- exact task list
- valid/invalid counts for both profiles
- extraction, compile, synthesis, and equivalence pass counts for both profiles
- failure category counts for both profiles
- per-task status for both profiles
- per-task generic cell counts where available
- per-task area-only scores where valid
- paired score delta for tasks valid under both profiles
- counts of v2 improved, matched, and regressed tasks versus neutral
- count of tasks where v2 gained or lost validity versus neutral
- explicit status for Stage M watchlist tasks: `adder`, `addr_calcu`, `calculation`, `divider_8bit`, and `comparator_16bit`
- mean valid score for each profile
- mean all-sample score for each profile
- recommendation: reject v2, revise v2, keep v2 as challenger, or promote v2 to full public RTL-OPT validation

Do not compute score deltas for invalid or unmatched tasks.

## 7. Decision Criteria

Recommend promoting `conservative_area_v2` to full public RTL-OPT validation only if:

- v2 has validity equal to or better than neutral, and
- v2 has mean all-sample score equal to or better than neutral, and
- v2 does not show widespread paired score regressions.

Recommend revising or rejecting v2 if:

- v2 loses validity versus neutral, or
- v2 has lower all-sample score, or
- v2 improves validity only by producing much larger generic-cell designs on paired-valid tasks.

## 8. Required Caveats

Reports must state:

- `generic_cells` is a public-flow area proxy, not final silicon area
- power metrics are unavailable
- timing is not being optimized or claimed
- this is still a public validation, not a SiliconCraft private benchmark

## 9. Success Criteria

Stage N succeeds if:

- Yosys preflight passes
- both profiles complete the same 40-task slice, or blocker is documented
- sanitized JSONL artifacts exist for both profiles if run executes
- summary Markdown/CSV exists for both profiles
- matched comparison Markdown/CSV exists
- no raw/private fields are committed
- report recommends the next action based on validity, all-sample score, and paired regressions

If no paired-valid tasks exist, the stage is still complete if the report clearly explains the blocker.

## 10. Commit Rules

Allowed:

- sanitized JSONL artifacts under `artifacts/v0.4/ppa_validation40/`
- Markdown/CSV reports
- docs/spec updates only if needed
- small report/export helper fixes only if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code from model runs
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 11. Non-goals

Do not:

- run full RTL-OPT
- run `low_area_low_power`
- run `conservative_area`
- run `strict_rtl_only`
- run VerilogEval, RTLLM, or ProtocolLLM
- run new models
- integrate SiliconCraft private benchmarks
- change v0.1/v0.2 frozen artifacts
- change v0.3 reports
- claim measured power improvement
- claim final silicon PPA quality

## 12. Definition of Done

Done when:

1. Preflight result is documented.
2. Exact 40-task list is documented.
3. Both profiles complete the matched run, or preflight/tooling blocker is clearly documented.
4. Sanitized PPA JSONL artifacts exist for executed profiles.
5. Summary Markdown/CSV reports exist.
6. Matched comparison Markdown/CSV reports exist.
7. Recommendation states whether v2 should be rejected, revised, kept as challenger, or promoted to full public RTL-OPT validation.
8. `python -m pytest` passes.
9. No raw outputs or model responses are committed.
