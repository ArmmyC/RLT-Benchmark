# Feature Spec: v0.7 RFID-APBench Post-Fix Expanded 10-Task Baseline

## Goal

Run a fresh expanded 10-task RFID-APBench baseline after the v0.7 response-boundary fix and `ap_001_idle_counter` prompt clarity update.

The post-clarity `ap_001` targeted validation passed 3/3 after the clarified prompt and fixed thinking-disabled runner path. The previous blocking issues are now addressed within targeted evidence:

- Qwen thinking-disable `extra_body` is forwarded by the RFID runner.
- The null-content token-limit symptom did not recur in targeted validation.
- The clarified `ap_001_idle_counter` prompt produced 3/3 functional passes in targeted validation.

This spec creates the next fixed-run public baseline across the current 10-task RFID-APBench manifest. It is not a model comparison, private evaluation, prompt-profile experiment, fine-tuning, or training-data creation effort.

## Required Scope

Run the current expanded RFID-APBench manifest baseline with:

- all current manifest tasks: exactly 10 tasks
- samples: exactly 3 samples per task, 30 attempted samples total
- model: configured/default `qwen36-27b` when available
- prompt profile: `neutral_baseline`
- max tokens: unchanged `4096`
- temperature/top-p: unchanged deterministic defaults
- thinking disabled: use fixed runner path with configured model preset `extra_body`
- benchmark assets: current public/synthetic assets, including the clarified `ap_001_idle_counter` prompt
- extractor/evaluator/scoring: unchanged
- committed outputs: sanitized reports only

If endpoint or tools are unavailable, do not fake results. Emit sanitized blocker/failure rows or a sanitized note.

## Required Files

Create:

## Report Requirements

The Markdown report must include:

- scope and non-goals
- source evidence used
- run configuration
- confirmation that the fixed thinking-disabled runner path was used
- confirmation that the current clarified `ap_001_idle_counter` prompt was included through the manifest
- endpoint and tool status without secrets
- task count, samples per task, and total attempted samples
- request outcome distribution
- empty/null-content/token-limit recurrence summary
- extraction/evaluation gate summary
- per-task aggregate summary
- valid-score count, mean valid score, and all-sample zero-filled score
- comparison against the earlier v0.6 expanded 10-task baseline, clearly labeled as different runner/prompt state and not a model comparison
- interpretation of whether v0.7 fixes improved baseline reliability and/or correctness
- benchmark asset decision
- fine-tuning readiness decision
- exactly one recommended next bounded direction
- activity proxy caveat
- public synthetic / no private / not fine-tuning caveat

## Decision Rules

Recommend exactly one:

- `v0.7_rfid_apbench_release_closeout_spec` if the fixed expanded baseline runs successfully and results are interpretable.
- `post_fix_failure_analysis_spec` if the run completes but shows important recurring invalid patterns requiring classification.
- `endpoint_or_tool_blocker_spec` if endpoint/tool availability blocks the run.
- `runner_bugfix_spec` if the fixed-run evidence reveals a runner/reporting/evaluation defect.
- `no_immediate_followup_beyond_documentation` if no further work is justified.

Do not recommend fine-tuning, private evaluation, model comparison, max-token increase, prompt-profile experiments, benchmark modification, or training-data creation from this baseline alone.

## Tests

Run:

```bash
python -m pytest
```

Add tests only if helper code is introduced. Do not require a live model endpoint in unit tests.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts beyond public benchmark prompt text
- raw model responses
- response bodies
- generated model RTL
- model outputs outside sanitized reports
- raw benchmark output directories
- VCD files
- simulator logs
- synthesis logs
- compiled artifacts
- tool scratch files
- secrets or endpoint credentials
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- add or modify benchmark tasks
- edit reference RTL or testbench behavior
- change module interfaces
- compare models
- change prompt profiles
- raise global max-token default
- create training data
- fine-tune a model
- add private benchmarks
- claim measured power
- claim final silicon PPA
- change workload, metrics, area/activity scoring, or evaluator policy

## Definition of Done

Done when:

1. Sanitized Markdown/CSV/JSONL post-fix expanded baseline reports exist.
2. The run attempts exactly 10 tasks × 3 samples, or endpoint/tool blockers are explicitly reported.
3. The report confirms the fixed thinking-disabled runner path and clarified `ap_001` prompt state.
4. Request/content/token metadata is summarized without raw outputs.
5. Evaluation gates and scores are reported.
6. Results are compared carefully against the earlier v0.6 expanded baseline without claiming a model comparison.
7. Benchmark assets remain unchanged during this run.
8. Fine-tuning is not recommended from this evidence alone.
9. Exactly one next bounded direction is recommended.
10. `python -m pytest` passes.
11. No raw/private/model-output/training artifacts are committed.
