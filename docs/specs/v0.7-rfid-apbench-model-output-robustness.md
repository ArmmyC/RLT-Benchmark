# Feature Spec: v0.7 RFID-APBench Model-Output Robustness Analysis

## Goal

Start v0.7 by analyzing model-output robustness issues observed during the expanded v0.6 RFID-APBench baseline, especially empty responses and incomplete/malformed outputs.

The v0.6 expanded closeout recommends `v0.7_model_output_robustness_spec` because the 10-task benchmark is validated, the expanded 30-sample model baseline is complete, benchmark assets remain unchanged, and the dominant invalid-row class is model-output reliability rather than benchmark/tool failure.

This is failure-taxonomy and planning analysis only. It is not prompt tuning, model comparison, fine-tuning, private evaluation, benchmark modification, or training-data creation.

## Inputs

Read first:

- `reports/v0.6_rfid_apbench_expanded_closeout.md`
- `reports/v0.6_rfid_apbench_expanded_baseline_analysis.md`
- `reports/v0.6_rfid_apbench_expanded_3sample_baseline.md`
- `reports/v0.6_rfid_apbench_expanded_3sample_baseline.csv`
- `reports/v0.6_rfid_apbench_expanded_3sample_baseline.jsonl`
- `reports/v0.6_rfid_apbench_post_tool_health_baseline_analysis.md`
- `reports/v0.6_rfid_apbench_3sample_baseline.md`
- `reports/v0.6_rfid_apbench_task_expansion_reference_validation.md`
- `reports/v0.6_rfid_apbench_task_expansion_candidate_smoke.md`

## Required Scope

Create a sanitized v0.7 planning/analysis report that:

1. Summarizes the model-output robustness evidence from v0.6.
2. Builds a high-level taxonomy of invalid model-output rows.
3. Separates empty response, incomplete/malformed response, candidate synthesis issue, and valid-output categories.
4. Identifies task-level concentration of empty/incomplete outputs.
5. States what evidence does and does not support endpoint instability, extractor defects, evaluator defects, or benchmark asset problems.
6. Recommends exactly one next bounded experiment or says no immediate experiment is needed.
7. Keeps benchmark assets unchanged unless strong evidence says otherwise.
8. Keeps fine-tuning out of scope.

## Required Files

Create:

- `reports/v0.7_rfid_apbench_model_output_robustness.md`

Optional:

- `reports/v0.7_rfid_apbench_model_output_robustness.csv` if a compact sanitized taxonomy table is useful
- helper script/tests only if helper code is introduced

Do not commit raw run directories, raw model responses, generated RTL, VCD files, simulator logs, synthesis logs, compiled artifacts, or tool scratch files.

## Analysis Requirements

The Markdown report must include:

- scope and non-goals
- source reports used
- v0.6 evidence summary
- invalid-row taxonomy
- task-level concentration table
- interpretation of empty responses
- interpretation of incomplete/malformed extraction failures
- interpretation of candidate-specific synthesis failure
- benchmark/tool/evaluator/endpoint assessment
- benchmark asset decision
- fine-tuning readiness decision
- exactly one recommended next step
- activity proxy caveat
- public synthetic / not private / not fine-tuning caveat

## Allowed High-Level Categories

Use sanitized categories such as:

- valid_output
- model_output_empty
- likely_incomplete_or_malformed_response
- likely_candidate_synthesis_issue
- endpoint_instability_not_observed
- extractor_bug_not_supported
- evaluator_bug_not_supported
- benchmark_asset_issue_not_supported
- insufficient_evidence

## Allowed Local Inspection

You may inspect ignored local/output artifacts only to classify failures at a high level, but committed text must stay sanitized.

Allowed committed descriptions:

- "empty response"
- "no complete required top module extracted"
- "candidate synthesis failed under healthy tools"
- "failure appears model-output related"
- "insufficient evidence from sanitized artifacts"

Forbidden committed content:

- raw generated RTL
- raw model response text
- raw prompt text beyond public file references
- simulator log excerpts
- synthesis log excerpts
- VCD snippets
- absolute output paths exposing raw artifacts
- private text or private RTL

## Decision Rules

Recommend exactly one:

- `bounded_prompt_profile_experiment_spec` if empty/incomplete outputs are concentrated enough to justify testing completion-reliability prompt/profile changes.
- `targeted_empty_response_investigation_spec` if the empty responses are task-specific and should be inspected before changing prompts.
- `runner_or_endpoint_observability_spec` if future runs need better sanitized metadata to distinguish empty model output from serving/runtime behavior.
- `no_immediate_experiment_beyond_tagging` if v0.7 should pause after taxonomy.

Do not recommend model comparison, private evaluation, training data creation, or fine-tuning from this analysis.

## Definition of Done

Done when:

1. `reports/v0.7_rfid_apbench_model_output_robustness.md` exists.
2. It summarizes v0.6 model-output robustness evidence clearly.
3. It classifies empty, incomplete/malformed, synthesis-failure, and valid-output rows at a sanitized high level.
4. It states whether benchmark assets remain unchanged.
5. It states fine-tuning is not recommended from this evidence alone.
6. It recommends exactly one next bounded step.
7. `python -m pytest` passes.
8. No raw/private/model-output/training artifacts are committed.
