# Feature Spec: v0.4 PPA-Aware RTL Optimization Benchmark

## 1. Goal

Implement the first code layer for correctness-gated PPA scoring without running new model jobs.

The v0.4 design docs are complete. They define the research question, correctness gates, area/power scoring formula, timing policy, missing-power behavior, invalid-score handling, sanitized schema, and the later `low_area_low_power` retest plan.

The next step is to implement pure scoring functions and unit tests only. Do not export real benchmark artifacts or run models in this stage.

## 2. Completed Design Inputs

Design docs:

- `docs/experiments/v0.4_ppa_aware_rtl_optimization.md`
- `reports/v0.4_ppa_scoring_design.md`

Key design decisions:

- v0.4 starts with public RTL-OPT before private SiliconCraft/RFID tasks.
- `qwen36-27b` remains the initial model.
- `neutral_baseline` remains default from v0.3.
- `low_area_low_power` is retested only after correctness-gated PPA scoring exists.
- Timing is a constraint/gate, not a reward.
- If trustworthy public power data is unavailable, use explicit area-only mode and never fake power results.

## 3. Required Scoring Contract

Required gates before optimization credit:

1. extraction
2. compile
3. synthesis
4. equivalence
5. timing, when required and available

If a required gate fails:

- per-sample view: `score_status = invalid`, `optimization_score = null`
- aggregate all-sample view: invalid sample contributes zero optimization credit

Full area/power score when all required metrics are available:

```text
score = 0.5 * area_score + 0.5 * power_score
area_score = min(reference_area / generated_area, 2.0)
power_score = min(reference_power / generated_power, 2.0)
```

Area-only mode when power is unavailable:

```text
scoring_mode = area_only
optimization_score = area_score
power_status = power_unavailable
power_score = null
```

Inputs must reject missing, zero, negative, non-finite, or unit-mismatched values for ratio computation.

## 4. Next Stage

### Stage B: Pure PPA scoring core

Status: next.

Implement pure Python scoring helpers that can be unit-tested without reading raw RTL, model responses, logs, or real benchmark outputs.

Suggested module:

- `src/rtlbench/ppa_scoring.py`

Suggested public API:

- data structures for gate outcomes and metric inputs
- function to determine first decisive failure category
- function to compute capped area ratio
- function to compute capped power ratio
- function to compute full area/power score
- function to compute area-only score
- function to produce a sanitized per-sample score record
- function to map invalid samples to zero for aggregate views

Exact names may differ, but the API must be simple, typed, deterministic, and tested.

## 5. Required Tests

Add tests for:

- valid full area/power scoring
- area score cap at 2.0
- power score cap at 2.0
- area-only mode with `power_unavailable`
- invalid extraction, compile, synthesis, equivalence, and timing gates
- missing/zero/negative/non-finite metric rejection
- invalid per-sample score is null
- invalid aggregate score maps to zero
- timing unavailable vs timing failure
- sanitized record contains no raw RTL, prompts, model responses, or logs

Suggested test file:

- `tests/test_ppa_scoring.py`

## 6. Commit Rules

Allowed:

- pure scoring module
- tests
- README/docs links or usage notes
- spec updates

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 7. Non-goals

Do not:

- run v0.4 model jobs
- run full benchmarks
- add new models
- implement export/summarize scripts yet, unless only as no-op stubs needed by tests
- change v0.1/v0.2 frozen artifacts
- change v0.3 reports
- claim PPA improvement without synthesis/equivalence evidence
- implement SiliconCraft private benchmarks yet

## 8. Definition of Done for Next Codex Task

The next task is done when:

1. Pure PPA scoring module exists.
2. Unit tests cover valid, capped, area-only, invalid, timing, and bad-metric cases.
3. Sanitized output schema excludes raw/private content.
4. `python -m pytest` passes.
5. No benchmark runs or raw outputs are committed.
6. The final summary explains what remains before retesting `low_area_low_power`.
