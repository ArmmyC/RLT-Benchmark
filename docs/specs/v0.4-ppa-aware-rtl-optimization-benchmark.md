## 4. Next Stage

### Stage I: PPA export generic-cell fallback fix

Status: next.

Fix the PPA export/metric handling so RTL-OPT records can use generic cell-count proxy metrics when mapped-area generated metrics are unavailable.

The exporter should not stop at mapped-area fields just because `reference_area` or `baseline_area` exists. It should choose the first complete valid metric pair by priority:

1. `reference_area` + `generated_area`
2. `baseline_area` + `generated_area`
3. `generic_baseline_cells` + `generated_cells`
4. otherwise invalid area metric

If mapped reference area exists but generated mapped area is missing, and generic cell metrics are complete, export a valid `area_only` record using:

- `reference_area = generic_baseline_cells`
- `generated_area = generated_cells`
- `area_unit = generic_cells`
- `toolchain_id = yosys-generic-cell-count-proxy`
- `scoring_mode = area_only`
- `power_status = power_unavailable`

Do not call generic cell-count proxy final silicon area.

## 5. Required Behavior

## 8. Commit Rules

Allowed:

- `src/rtlbench/ppa_export.py`
- tests for PPA export fallback
- sanitized fixed-control JSONL/report updates
- docs/spec updates only if needed
- small evaluator/runner metric persistence fix if required

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`

- prompts containing private tasks
