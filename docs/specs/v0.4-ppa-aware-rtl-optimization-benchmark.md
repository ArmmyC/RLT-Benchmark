# Feature Spec: v0.4 PPA-Aware RTL Optimization Benchmark

## 1. Goal

Add a sanitized v0.4 export layer that converts structured RTL-OPT result rows into PPA score records.

The v0.4 design docs, pure scoring core, and sanitized reporting layer are complete. The next layer should bridge existing RTLBench result rows to the v0.4 scoring/reporting contract without running new model jobs and without committing raw outputs.

## 2. Completed Inputs

Design docs:

- `docs/experiments/v0.4_ppa_aware_rtl_optimization.md`
- `reports/v0.4_ppa_scoring_design.md`

Scoring core:

- `src/rtlbench/ppa_scoring.py`
- `tests/test_ppa_scoring.py`

Reporting layer:

- `src/rtlbench/ppa_reporting.py`
- `scripts/summarize_v0.4_ppa_results.py`
- `tests/test_ppa_reporting.py`

Existing RTLBench run rows include structured pass/fail fields and `evaluation_metrics`, but they may also contain paths to raw responses, extracted RTL, and logs. The v0.4 export layer must never copy those raw/private path fields into sanitized PPA artifacts.

## 3. Next Stage

### Stage D: RTL-OPT result-row to sanitized PPA export

Status: next.

Implement conversion from already-structured RTLBench result rows to sanitized v0.4 PPA score records.

Suggested module:

- `src/rtlbench/ppa_export.py`

Suggested script:

- `scripts/export_v0.4_ppa_results.py`

The script may read `results.jsonl` files produced by RTLBench, but must read only structured JSON rows. It must not open raw response files, extracted RTL files, error logs, Yosys logs, prompts, or benchmark source RTL files.

No model jobs should be run in this task.

## 4. Conversion Rules

For initial public RTL-OPT support:

- only support `benchmark == "rtlopt"`
- use `model` from the result row
- accept an explicit `--prompt-profile` argument or derive it from run metadata if safely available
- use task/sample identifiers from the result row
- use `evaluation_metrics` for area-like metrics
- use area-only mode unless trustworthy power fields are explicitly present
- set `power_status = power_unavailable` when power is unavailable
- do not infer power from area, cell count, or prompt text

Recommended area source priority:

1. `reference_area` and `generated_area` when both are available and valid
2. `baseline_area` and `generated_area` when reference area is unavailable
3. generic cell proxy using `generic_baseline_cells` and `generated_cells` when mapped area is unavailable

When generic cells are used as the area proxy:

- set `area_unit = generic_cells`
- clearly identify the toolchain as a generic Yosys/cell-count proxy
- do not call this final silicon area

Gate mapping:

- extraction passes when an extracted RTL path exists or the failure category is not `code_extraction_failure`
- compile pass comes from `compile_pass`
- synthesis pass requires compile pass and suitable generated area/cell metric when using PPA export
- equivalence pass should use `final_pass` or `evaluation_metrics.equivalence_proven == 1` for RTL-OPT equivalence rows
- timing should be `not_required` or `unavailable` unless real timing data is present and declared required

Failure handling:

- failed extraction, compile, synthesis, equivalence, or timing gates produce invalid score records
- missing/invalid area metrics produce invalid score records
- invalid records must keep `optimization_score = null`
- aggregate views later map invalid records to zero

## 5. Sanitization Requirements

The exported JSONL must contain only fields accepted by `PPAScoreRecord`.

It must not contain:

- raw response paths
- extracted RTL paths
- error log paths
- log/work directory paths
- prompts
- raw model responses
- generated RTL
- source RTL
- API keys
- private task text

## 6. Required Tests

Add tests for:

- converting a valid RTL-OPT equivalence row with mapped area
- converting a valid generic cell-count row in area-only mode
- compile failure becomes invalid score
- equivalence failure becomes invalid score
- missing metric becomes invalid score
- forbidden raw path fields are not copied to sanitized output
- unsupported benchmark is rejected or skipped with explicit reason
- export script reads synthetic `results.jsonl` and writes sanitized PPA JSONL
- exported JSONL can be summarized by `scripts/summarize_v0.4_ppa_results.py`

Suggested test file:

- `tests/test_ppa_export.py`

Use synthetic structured result rows only. Do not use real run output directories or raw artifacts in tests.

## 7. Commit Rules

Allowed:

- PPA export module
- PPA export script
- tests using sanitized synthetic fixtures
- README/docs usage notes
- spec updates

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 8. Non-goals

Do not:

- run v0.4 model jobs
- run full benchmarks
- read raw response files or extracted RTL files
- parse Yosys logs directly in this stage
- integrate with SiliconCraft private benchmarks
- add new models
- change v0.1/v0.2 frozen artifacts
- change v0.3 reports
- claim PPA improvement

## 9. Definition of Done for Next Codex Task

The next task is done when:

1. RTL-OPT PPA export module exists.
2. Export script exists and defaults to sanitized JSONL output.
3. Tests cover valid mapped-area, generic-cell, invalid, missing-metric, and unsupported cases.
4. Export strips or rejects raw/private fields.
5. Exported synthetic JSONL can be summarized by the v0.4 PPA summary script.
6. `python -m pytest` passes.
7. No benchmark runs or raw outputs are committed.
8. The final summary explains what remains before running a small v0.4 control validation.
