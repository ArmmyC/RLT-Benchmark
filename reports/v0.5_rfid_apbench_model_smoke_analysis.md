## `ap_001_idle_counter` Extraction Failure

`ap_001_idle_counter` completed the generation step but failed extraction, so no candidate file was emitted and the evaluator reported `candidate_missing`. A bounded inspection of ignored local response artifacts found two safe, high-level signals:

- In the committed real smoke run, the ignored response artifact for `ap_001_idle_counter` contained zero text.
- In the earlier aborted report-writer run, the same task had a required-top module start but no complete module terminator, so the existing extractor correctly rejected it.

The extractor requires a complete module block and, when a required top is supplied, a matching top-module declaration. The evidence does not point to a task naming mismatch: the required top name was present in the earlier ignored artifact. It also does not point to a compile/simulation/synthesis issue, because no extracted candidate reached those gates. The most likely category is model response behavior: empty or incomplete output for the first task under the bounded smoke settings. A secondary possibility is endpoint/serving behavior causing an empty completion for that sample; the sanitized row itself does not expose enough detail to separate model silence from serving-side truncation.

## Recommended Bounded Retry Plan

Recommend Option A: retry `ap_001_idle_counter` only with the same model, endpoint, prompt profile, temperature, top-p, max token limit, and evaluator settings for 3 samples.

Rationale:

- Only one task failed extraction.
- The other four tasks extracted and evaluated successfully in the same run.
- The failed task evidence points to empty or incomplete model response behavior, not a benchmark-wide evaluator failure.
- A 3-sample targeted retry is enough to distinguish a one-off empty/incomplete response from a repeatable task-specific generation issue without creating a broad prompt-tuning campaign.

Retry acceptance criteria:

- Keep raw responses, extracted RTL, generated candidates, logs, VCDs, and synthesis scratch under ignored output/work directories.
- Report only sanitized per-sample extraction, compile, correctness, synthesis, area, activity, and score fields.
- If at least one retry sample extracts and reaches evaluation, compare it only against the same task/reference metrics.
- If all three retry samples fail extraction, classify `ap_001_idle_counter` as repeatably extraction-fragile for this model/settings pair and defer prompt or extractor changes to a separate scoped proposal.
