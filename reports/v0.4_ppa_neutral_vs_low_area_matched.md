# v0.4 Neutral vs Low-Area Matched PPA Retest

This report compares the fixed `neutral_baseline` control against a matched `low_area_low_power` retest on the same five public RTL-OPT tasks.

## Scope

- Benchmark: `rtlopt`
- Model: `qwen36-27b`
- Samples per task: 1
- Evaluator: `yosys_equiv`
- Scoring mode: `area_only`
- Area unit: `generic_cells`
- Toolchain ID: `yosys-generic-cell-count-proxy`
- Power: unavailable

The Yosys preflight passed before the low-area model calls:

```text
PASS techmap_mode=near_yosys_executable
```

The matched task slice was:

- `add_sub`
- `adder`
- `adder_carry`
- `adder_select`
- `addr_calcu`

No full RTL-OPT, VerilogEval, RTLLM, ProtocolLLM, `strict_rtl_only`, new-model, or private-task run was executed.

## Paired Outcome

Among paired-valid tasks:

- Improved: 0
- Matched: 2
- Regressed: 0
- Paired-valid tasks: 2

On the paired-valid subset, `low_area_low_power` matched `neutral_baseline` under the generic-cell proxy score. It did not improve the paired score in this five-task slice, and it had one fewer valid correctness-gated record overall because `add_sub` failed equivalence under `low_area_low_power`.

## Caveats

`generic_cells` is a public-flow proxy for area-only optimization scoring. It is not final silicon area and must not be reported as final silicon PPA quality.

Power is unavailable in this flow. This report makes no measured power-improvement claim.
