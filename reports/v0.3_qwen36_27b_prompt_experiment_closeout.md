# v0.3 qwen36-27b Prompt Experiment Closeout

This report closes out the v0.3 prompt-profile experiment for `qwen36-27b`.

The experiment compared prompt strategies on the same model. It was not a model comparison, not a full benchmark rerun, and not a final silicon PPA study.

## Final Recommendation

Keep `neutral_baseline` as the default prompt profile for daily `qwen36-27b` RTL experiments.

Keep `strict_rtl_only` as an optional challenger profile for future experiments.

Do not promote `low_area_low_power` yet.

Do not promote `no_explanation_code_only`.

The larger validation tied at 31/40 for `neutral_baseline` and `strict_rtl_only`. `strict_rtl_only` was slightly better on VerilogEval, `neutral_baseline` was slightly better on RTL-OPT equivalence, and RTLLM was tied. There is no clear evidence to switch the default prompt.

## Experiment Goal

The goal of v0.3 was to test whether prompt-profile changes improve RTL generation behavior for `qwen36-27b` while keeping the model fixed and preserving matched benchmark settings.

The experiment focused on functional RTL behavior:

- VerilogEval functional simulation
- RTLLM functional simulation
- RTL-OPT equivalence

## Prompt Profiles Tested

| Prompt profile | Role | Final status |
| --- | --- | --- |
| `neutral_baseline` | Historical/control prompt | Default for now |
| `strict_rtl_only` | Stricter RTL-only formatting prompt | Optional challenger |
| `low_area_low_power` | Area/power-oriented prompt hypothesis | Held back |
| `no_explanation_code_only` | Strict code-only formatting prompt | Held back |

## Smoke Results

The smoke matrix used 3 samples per benchmark/profile across VerilogEval, RTLLM, and RTL-OPT.

| Prompt profile | Smoke total | Smoke interpretation |
| --- | ---: | --- |
| `neutral_baseline` | 7/9 | Selected for larger validation |
| `strict_rtl_only` | 7/9 | Selected for larger validation |
| `low_area_low_power` | 6/9 | Held back because RTL-OPT equivalence smoke was weaker |
| `no_explanation_code_only` | 5/9 | Held back because it was weakest overall and had an RTLLM compile failure |

The smoke stage showed that `neutral_baseline` and `strict_rtl_only` were the strongest practical candidates. The other two profiles remain useful hypotheses, but neither earned promotion from the smoke screen.

## Candidate Decision

The candidate decision selected only:

- `neutral_baseline`
- `strict_rtl_only`

for larger validation.

It held back:

- `low_area_low_power`
- `no_explanation_code_only`

The decision was intentionally conservative. The low-area/low-power profile should not be treated as evidence of PPA improvement without synthesis/PPA reports and equivalence-preserving analysis.

## Larger Validation Results

The larger validation compared the two selected candidates only.

| Prompt profile | VerilogEval | RTLLM | RTL-OPT equivalence | Total |
| --- | ---: | ---: | ---: | ---: |
| `neutral_baseline` | 19/20 | 4/10 | 8/10 | 31/40 |
| `strict_rtl_only` | 20/20 | 4/10 | 7/10 | 31/40 |

Interpretation:

- `strict_rtl_only` was slightly better on VerilogEval.
- `neutral_baseline` was slightly better on RTL-OPT equivalence.
- RTLLM was tied.
- The total result was tied, so there is no clear winner.

## Limitations

The larger validation slice is larger than smoke, but still smaller than a full benchmark sweep.

The experiment used `qwen36-27b` only. It does not compare models.

The experiment does not establish final silicon PPA quality. Area, power, and timing claims require dedicated synthesis/PPA evidence and equivalence-preserving analysis.

The prompt conclusions are scoped to the tested benchmarks, task slices, settings, and model endpoint.
