# v0.4 PPA Control Before/After

This report compares the first five-task public RTL-OPT `neutral_baseline` PPA control against the Stage H rerun after the Yosys techmap fix. It uses sanitized v0.4 PPA score records for the committed before/after counts.

## Run Scope

Both runs used the same control slice:

- Model: `qwen36-27b`
- Benchmark: `rtlopt`
- Prompt profile: `neutral_baseline`
- Limit: 5 tasks
- Samples per task: 1
- Evaluator: `yosys_equiv`
- Scoring mode: `area_only`

The Stage H preflight passed before the rerun:

```text
PASS techmap_mode=near_yosys_executable
```

No `low_area_low_power`, `strict_rtl_only`, full benchmark, new model, or private-task run was executed.

The Stage H rerun demonstrates that the Yosys techmap path fix removed the previous toolchain lookup blocker. It does not yet produce valid v0.4 PPA score records because the committed sanitized PPA schema scores mapped area, and `generated_area` is still null in all five fixed records.

## Decision

`low_area_low_power` remains blocked.

The next v0.4 step should fix or configure the PPA export/evaluation path so the neutral control produces at least one valid correctness-gated PPA record with generated area. Only after that neutral control succeeds should a matched `neutral_baseline` versus `low_area_low_power` retest be scheduled.
