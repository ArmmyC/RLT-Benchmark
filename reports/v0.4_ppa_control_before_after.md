# v0.4 PPA Control Before/After

This report compares the first five-task public RTL-OPT `neutral_baseline` PPA control against the Stage H rerun after the Yosys techmap fix. It uses sanitized v0.4 PPA score records for the committed before/after counts.

## Run Scope

Both runs used the same control slice:

- Model: `qwen36-27b`
- Benchmark: `rtlopt`
- Prompt profile: `neutral_baseline`
- Limit: 5 tasks
- Samples per task: 1
- Evaluator: `yosys_equiv`
- Scoring mode: `area_only`

The Stage H preflight passed before the rerun:

```text
PASS techmap_mode=near_yosys_executable
```

No `low_area_low_power`, `strict_rtl_only`, full benchmark, new model, or private-task run was executed.

## Failure Categories

| Run | Sanitized failure categories |
| --- | --- |
| Previous control | `compile_failure=5` |
| Fixed control | `passed=3, equiv_failure=2` |

The failure category changed because the fixed evaluator now gets through the Yosys compile/generic synthesis entry point that previously failed on `techmap.v` lookup. The rerun reaches equivalence for three samples. After the Stage I exporter fallback fix, those three equivalence-passing samples are valid area-only records using the generic cell-count proxy.

| Timing metrics | not required | not required |

The Stage H rerun demonstrates that the Yosys techmap path fix removed the previous toolchain lookup blocker. The Stage I exporter fallback fix now records complete generic cell-count proxy metrics as `area_unit=generic_cells` with `toolchain_id=yosys-generic-cell-count-proxy` when mapped generated area is unavailable.

These generic cell counts are a public-flow proxy for area-only optimization scoring. They are not final silicon area, they do not include power, and they must not be reported as final PPA quality.

## Decision

`low_area_low_power` can be scheduled next as a matched `neutral_baseline` versus `low_area_low_power` experiment on the same public RTL-OPT slice, using the same generic-cell proxy scoring caveats.

The next run should still avoid power or silicon-quality claims unless a later flow adds measured power and a mapped timing/power methodology.
