- Evaluator: `yosys_equiv`

## Conservative vs Neutral

Among paired-valid tasks:

- Improved: 0
- Matched: 3
- Regressed: 0
- Paired-valid tasks: 3

`conservative_area` matched `neutral_baseline` on all three paired-valid tasks. It did not improve the generic-cell proxy score, but it also did not reduce correctness coverage relative to neutral on this five-task slice.

## Conservative vs Low-Area Validity

`conservative_area` improved validity relative to `low_area_low_power`: 3 valid records versus 2. The recovered task was `add_sub`, where `low_area_low_power` failed equivalence but `conservative_area` passed.

## Recommendation

Keep `conservative_area` as a safer challenger profile for the next small matched experiment, but do not promote it over `neutral_baseline` yet. In this slice it restored neutral-level correctness coverage and avoided the validity loss seen with `low_area_low_power`, but it did not produce a paired generic-cell score improvement over neutral.

The existing `low_area_low_power` profile should remain unscaled until it is revised or a stronger matched result appears.

## Caveats

`generic_cells` is a public-flow proxy for area-only optimization scoring. It is not final silicon area and must not be reported as final silicon PPA quality.

Power is unavailable in this flow. This report makes no measured power-improvement claim.
