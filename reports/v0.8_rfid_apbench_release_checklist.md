## How To Use Before Tagging

Before tagging, copy `docs/release/rfid_apbench_release_checklist.md` into the release issue, PR, or release notes draft. Check each item against current repository evidence, then run the local verification commands listed in the checklist. The release owner should confirm that tests pass, benchmark/config assets are unchanged except approved public benchmark changes, prohibited artifacts are absent from the tracked diff, and release claims stay within the evidence.

Only after the checklist is reviewed and complete should the release owner manually run the intended tag commands.
