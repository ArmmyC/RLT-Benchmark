# v0.7 RFID-APBench Bounded Prompt-Profile Experiment

## Scope

This bounded experiment tests whether one generic completion-reliability system profile reduces successful empty/null-content responses on the two public/synthetic RFID-APBench tasks where empty output was concentrated:

- `ap_001_idle_counter`
- `ap_010_retry_timeout_fsm`

The control is `neutral_baseline`. The challenger is the additive `completion_reliability` profile, which requests one concise, complete RTL module without prose, Markdown fences, hidden reasoning, or lengthy explanation and requires a final `endmodule` within the token budget.

Exactly three samples were attempted per task per profile: 6 samples per profile and 12 samples total. Benchmark task prompts, references, testbenches, workloads, metrics, extractor, evaluator, scoring, model, and generation settings were unchanged.

## Non-Goals

- No benchmark task or task prompt was added or modified.
- No model comparison, benchmark-wide baseline, private evaluation, training-data creation, fine-tuning, or adapter work was performed.
- No measured power, signoff power, final silicon PPA, or production QoR claim is made.
- No raw prompt beyond the public profile text, raw response, response body, generated RTL, VCD, log, compiled artifact, tool scratch, secret, private data, model weight, dataset, or adapter is committed.

## Interpretation

The completion-reliability challenger did not reduce successful empty/null-content responses. Its 5/6 overall empty rate, per-task concentration, empty-row metadata, downstream gates, and failure distribution are identical to the fresh neutral control. It introduced no distinct extraction or syntax regression, but it also provided no output-reliability or correctness benefit.

This bounded deterministic result rejects the tested profile for follow-on validation. It does not establish that prompt profiles can never help, compare different models, or diagnose hidden server-side generation behavior beyond the sanitized metadata.

## Benchmark Asset Decision

Keep benchmark assets unchanged. Reference validation and candidate-fixture evidence remain healthy, and the matched profile outcomes provide no evidence of a defect in either task prompt, reference, testbench, workload, extractor, evaluator, metric, or scoring rule.

## Fine-Tuning Readiness

Fine-tuning remains out of scope and is not recommended from this limited public/synthetic evidence. The experiment creates no training dataset and does not provide sufficient causal or governance evidence for model-weight or adapter changes.

## Recommended Next Step

Recommend exactly one next bounded step: `profile_rejected_closeout_spec`.

This follows the decision rule because the challenger did not improve empty-response behavior and did not improve downstream gates. No additional prompt-profile validation is justified for this challenger.
