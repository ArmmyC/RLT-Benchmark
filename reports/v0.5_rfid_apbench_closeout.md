# v0.5 RFID-APBench Closeout

## Scope

This closeout summarizes the v0.5 public/synthetic RFID-APBench benchmark state and records the handoff to the v0.6 roadmap.

RFID-APBench activity is a VCD toggle-count proxy. It is not measured silicon power, signoff power, final silicon PPA, or deployed RFID/NFC energy.

## Non-Goals

- No benchmark task, prompt, reference RTL, testbench, extractor, evaluator, or model-smoke script changes are made by this closeout.
- No model jobs are run by this closeout.
- No private RTL, private task text, raw prompts, raw model responses, generated RTL, raw outputs, logs, VCDs, secrets, model weights, training datasets, LoRA/QLoRA/DoRA adapters, or fine-tuning scripts are committed.

## Current Benchmark Limitations

- v0.5 has only five public/synthetic tasks.
- The first model smoke used one sample per task.
- The only repeated-sample run targeted one failed task.
- `ap_001_idle_counter` has no valid model retry score.
- Failure taxonomy is currently task-specific and should be broadened in v0.6.
- Sanitized model-evaluation records exist in reports, but a stable v0.6 record contract should be formalized.
- Activity is a VCD toggle-count proxy, not measured silicon power.

## Why Benchmark Assets Remain Unchanged

Benchmark assets remain unchanged because:

- reference validation passed 5/5
- public candidate fixture smoke passed 5/5
- `ap_001_idle_counter` public prompt, reference, and testbench are internally consistent
- extracted idle-counter retry candidates compiled and synthesized, indicating no top-module/interface or extractor truncation issue
- the observed idle-counter failures are model-produced behavioral mismatches, not evidence of a benchmark asset defect

No task text, prompt, reference RTL, testbench, extractor, evaluator, or model script change is justified by v0.5 evidence.

## Why Fine-Tuning Is Not Ready

Fine-tuning is not ready because RFID-APBench is still an evaluation MVP, not a training-data source.

Blocking reasons:

- only five public/synthetic tasks exist
- no 3-sample baseline exists across all five tasks
- `ap_001_idle_counter` produced no valid retry score
- failure taxonomy needs a broader multi-sample baseline
- stable sanitized model-evaluation record contracts still need v0.6 hardening
- private RTL and training data remain explicitly out of scope

No fine-tuning, prompt tuning campaign, extractor bugfix, task change, private evaluation, or training repository work should start from this single v0.5 closeout.

## v0.6 Handoff

The v0.6 roadmap should focus on:

- 3-sample baseline on the existing five public/synthetic tasks
- expansion to 10-15 public/synthetic RFID/AP tasks
- failure taxonomy reporting
- stable sanitized model-evaluation records
- continued exclusion of private RTL, private data, training data, and fine-tuning artifacts

The companion roadmap spec is `docs/specs/v0.5-rfid-apbench-closeout-and-v0.6-roadmap.md`.
